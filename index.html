Robots!

This past summer while working at BNY Mellon, I took an Honors course through Pitt's CS department aimed at writing code in a real environment. We were assigned a partner, a laptop, and a Kobuki (which is essentially a rumba that we could manipulate through code). We learned python and the ROS (Robotic operating system framework) framework, and used these tools to create algorithms to complete three projects. The first project was simple : Command the robot to move an exact distance forward and backwards, or turn to an exact angle. This was accomplished through the use of the robot's build in odometer. This project required us to create a velocity smoother, which would ramp up or slow down the robot upon starting or stopping. Without using a smoother, the robot would jolt as it began to change speeds rapidly, cause the odometer to give incorrect data. The second project was a line follower, for which we used a color detection library. The third project was to make the robot autonomously travel through the hallways of one of the floors in the CS building. This required a very creative solution and A LOT of testing. For this project we were given an XBox Kinect, which we used for distance detection. The easy part of this was to travel down a straight hallway. We simply balanced the distance of the object (wall) to the left and right of the robot, turning slightly to make sure that these were always approximately even. The difficult part was to have the robot turn corners. To do this, we created a simple classification algorithm for detecting a turn, and whether it was a right or left turn. Then, upon coming close enough to the corner, the robot turned at a constant speed (while moving forward) until it had passed the corner.
